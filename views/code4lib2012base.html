NoSQL Bibliographic Records: Implementing a Native FRBR Datastore with Redis

Jeremy Nelson
Colorado College
Tutt Library

Begin »

Background

The FRBR-to-Redis-Datastore project depends on a basic understanding of the following biblographic record technologies: MARC, MODS, and FRBR.

The October 2011 Library of Congress annoucement regarding a Bibliographic Framework for the Digital Age and a NoSQL technology called Redis provide the development requirements and datastore implmentation for this project.

View details »

Record to Cube

The underlying concept of the FRBR Redis Datastore is the Biblographic Cube. Instead of trying to creating a single, flat a canonical biblographic record; the datastore orchastractes a network of key-value biblographic associations with FRBR entities.

Better visualized as moving from a flat record (Dublin Core, MARC21, MODS XML, etc.) to a cube of FRBR entities interseting with parallel planes of metadata.

View details »

Engineering

Redis's set of data primitives including sets, hashes, and lists implemented as serialized caching objects provided the datastore for representing FRBR entities.

The rich set of work done by the RDA Vocabularie communitie including their SKOS representations of FRBR RDA provides the organizing structure for the FRBR entities.

Intially focusing on interoperability for existing MARC collections of records captured by legacy ILS systems and with current repository systems with MODS metadata. Other metadata schemas mentioned are implementation stages.
