Background

The FRBR-to-Redis-Datastore platform project begins with a basic understanding of the following bibliographic record technologies MARC, MODS, and FRBR implemented using a NoSQL technology called Redis.

With a recent Library of Congress annoucement, Bibliographic Framework for the Digital Age as software development requirements The FRBR-to-Redis-Datastore platform project a proof-of-concept using the MARC catalog and digital repository of a small academic library.

Record to Cube

The underlying concept of the FRBR Redis Datastore is the Bibliographic Cube. Instead of trying to creating a single, flat a canonical bibliographic record; the datastore orchestrates a network of key-value bibliographic associations with FRBR entities.

Better visualized as moving from a flat record (Dublin Core, MARC21, MODS XML, etc.) to a cube of FRBR entities intersecting with parallel planes of metadata branes.

Engineering

The ${project.name} Redis's set of data primitives including sets, hashes, and lists implemented as serialized caching objects provided the datastore for representing FRBR entities.

The rich set of work done by the RDA Vocabularies communities including their SKOS representations of FRBR RDA provides the organizing structure for the FRBR entities.

Initially focusing on interoperability for existing MARC collections of records captured by legacy ILS systems and with current repository systems with MODS metadata. Other metadata schemes mentioned are implementation stages.
