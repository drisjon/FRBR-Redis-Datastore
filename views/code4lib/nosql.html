NoSQL & Redis

NoSQL is a class of newer data storage technologies that sidestep traditional normalized databases that use SQL and schemas. While a comprehensive review of NoSQL projects was not done, investigation was done on a few commercial and open-source NoSQL projects when Redis was discovered and work was started on ${project.name} as a distinct project. Redis was selected for its easy of deployment and flexibility in exploring new data structures for representing bibliographic records.

## Google Bigtable and Hadoop

BigTable, Google App Engine, and Hadoop

  • Bigtable is the original distributed storage system for managing structured data at Google and the inspiration of Hadoop and Cassandra NoSQL open-source projects.
  • Google App Engine provides access to Google proprietary implementation of BigTable and GFS but is limited by restrictions and is tied to Google's Cloud.
  • Hadoop is one of the fastest growing open-source implementation of BigTable. It can be complex to set-up; but would seem to work for well for an FRBR-based datastore

Other NoSQL

There are many other NoSQL datastore technologies including document stores like CouchDB and MongoDB and more well-known graph datastores FlockDB and Neo4j.

Facebook donated their Cassandra technology to the Apache foundation, that along with Amazon's Dynamo are examples of eventually‐consistent key‐value stores.

Why Redis?

The key-value Redis.io nature means Redis is fast, small, and easy to deploy; key requirements of the LOC bibliographic frameworks and crucial for smaller libraries and leaner cultural IT organizations.

  • Fast, C code
  • Multiple language bindings including Python, Ruby, PHP, Perl, C#, and Java.
  • Easy to represent multi-dimensional nature of bibliographic records
